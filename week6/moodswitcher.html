Project 7--DOM and JS

JavaScript Questions

  1. 1. Why is textContent safer than innerHTML?
  2. Answer: textContent only handles the text within and not the HTML itself. When using innerHTML changes occur to the HTML tags as well. If entering information from the user it is more logical to expect them to provide the text they want to change without the HTML. Security wise it is also better as it presents fewer vulnerablities to exploit if only text can change.
  3. 2. How are HTML elements connected to JavaScript variables using DOM selection?
  4. Answer: In the DOM there are 12 node types. In the head of the HTML we can call a JavaScript document that contains our JavaScript (.js). Within that document, nodes can be called with the document object that you want to target. You can use tags, ids, labels and other identifiers to call specific items. The most common ways we will be using to reference these elements in JavaScript to make changes is using document.querySelector, document.querySelectorAll, and document.getElementById.
  5. What would happen to the text box with id="text" if I added this line to the code above it: text.textContent = "Hello!"
  6. Answer: The text within the tag with the id of text would be replaced or populated with Hello! .
