  1. Name a real-life situation where you would need to use an `if` statement.
  2. A situation in which an 'if' statement may be used is in grading. That situation would utilize else if and would provide a letter grade based on the percentage of a student's score. For example, a 90% or better is an A, an 80% or better is a B, etc...
  3. Explain the difference between `==` and `===`. Why is strict equality usually better?
  4. Loose equality (==) is used to compare two values but allows JavaScript to try to do the type matching itself. So if you compare "18" and 18 (one a string and one a number) it will convert the type of the string to a number to do the comparison. This may not provide the most reliable results as when dealing with edge cases regarding comparing 0s, Booleans, NaN, Undefined and Nulls. Best practice to set the type yourself and use Strict equality (===) which requires equality of type and content to give true as a result, and a mismatch of either results in false. This provides accuracy in your comparison.
  5. Give an example (in your own words) of when you would use `&&` (AND) vs `||` (OR).
  6. Using && or || is required when you have multiple variables you are looking at in order to provide a result. && requires both statements to be true in order to give a true result while || requires only one of the statements to be true to give a true result. For example, as an employee when you log in at work your computer should first check if you are a current employee and then verify that the user/password you entered is correct. Only if both are true (&&) should you be allowed access. An instance when you might use || is when going to the movies. R rated movies require someone be 18 or accompanied by an adult to attend. In that case you have two comparisons and if either are true the person can attend their preferred show.
  7. What does the `!` (NOT) operator do? Give an example of a situation you would use it.
  8. The ! operator is essentially the opposite of equals. It provides a comparison if you are looking for a exclusion instead of inclusion of your example. In my view == or === are inclusion type comparisons, so if what you are comparing does match then do this. != or !== are exclusion based comparisons so if what you are comparing does not match then do this. The difference is mainly in what gives you a true response. To my thoughts the most common use would be if an answer is valid. If you are comparing two variables the first thing I would do is validate that the variables are comparable. An example of this would be looking for non answers in a JavaScript prompt. So if I was looking to compare two numbers, and someone typed out their number (nine instead of 9), my first thing to evaluate is if the answer type is != to number. If it is not a number then I give an error message and if it is I can continue onto the number comparison I am doing.
